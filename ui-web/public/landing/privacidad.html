Responsable del tratamiento

El responsable del tratamiento de los datos personales recopilados a través de Inteliaudit es:

  • Empresa: Intellihouse
  • Domicilio: Pedro Juan Caballero, Amambay, República del Paraguay
  • Email de contacto: hola@inteliaudit.com.py

Intellihouse es la entidad que determina los fines y los medios del tratamiento de los datos recopilados directamente de los usuarios de la plataforma (nombre, email, datos de cuenta). Para los datos de contribuyentes procesados como parte del servicio de auditoría, Intellihouse actúa como procesador por cuenta del usuario auditor.

2.2 Datos procesados como parte del servicio de auditoría

Durante el uso del servicio, el usuario carga o genera datos de los contribuyentes que audita: declaraciones juradas (Form. 120, 500, etc.), archivos RG90, datos de SIFEN y HECHAUKA, comprobantes y registros contables. Estos datos son procesados por Inteliaudit exclusivamente para prestar el servicio contratado y pertenecen al usuario.

Credenciales Marangatú: Las credenciales de acceso al portal SET (RUC y clave) se almacenan siempre encriptadas con AES-256. Nunca se almacenan en texto plano. El acceso a estas credenciales está restringido al proceso automatizado de descarga de datos y al propio usuario.

3

Finalidad del tratamiento

Los datos recopilados se utilizan exclusivamente para las siguientes finalidades:

  • Prestación del servicio de auditoría impositiva automatizada contratado por el usuario.
  • Autenticación y gestión de la cuenta del usuario en la plataforma.
  • Envío de notificaciones relacionadas con el servicio (alertas de auditoría, actualizaciones, vencimientos).
  • Facturación y gestión de la suscripción.
  • Soporte técnico ante solicitudes del usuario.
  • Mejora del servicio mediante análisis agregados y anónimos de uso de la plataforma.

Los datos nunca son utilizados para elaborar perfiles con fines comerciales ajenos al servicio, ni para publicidad dirigida a terceros.

4

Almacenamiento y seguridad

Inteliaudit implementa las siguientes medidas técnicas y organizativas para proteger los datos:

  • Encriptación en reposo: Todos los datos sensibles (credenciales, datos de auditoría) se almacenan encriptados con AES-256.
  • Encriptación en tránsito: Todas las comunicaciones entre el navegador del usuario y los servidores de Inteliaudit se realizan mediante HTTPS/TLS 1.3. No se acepta tráfico HTTP sin cifrar.
  • Control de acceso: El acceso a los datos de producción está restringido al personal técnico estrictamente necesario, bajo autenticación multifactor.
  • Audit trail: Todas las acciones realizadas sobre datos de auditoría quedan registradas con marca de tiempo, usuario y dirección IP. El registro no puede modificarse ni eliminarse.
  • Backups: Los datos se respaldan automáticamente con frecuencia diaria y se almacenan de forma segura con retención de 30 días.
  • Pruebas de seguridad: La plataforma se somete a revisiones periódicas de seguridad.
5

Compartición de datos con terceros

Compromisos principales: Inteliaudit NO vende, NO cede y NO comparte los datos del usuario ni de los contribuyentes auditados con terceros para fines comerciales propios o ajenos.

Los únicos terceros que pueden tener acceso a datos son los proveedores de infraestructura tecnológica necesarios para la operación del servicio:

  • Proveedor de hosting / servidores: Para el almacenamiento y procesamiento de datos en la nube. El proveedor opera bajo acuerdo de confidencialidad y procesamiento de datos.
  • Proveedor de base de datos: Para el almacenamiento estructurado de información. Mismo régimen de confidencialidad.
  • Proveedor de pagos: Únicamente recibe los datos de facturación necesarios para procesar la transacción. No accede a datos de auditoría.

Todos los proveedores de infraestructura son seleccionados por sus certificaciones de seguridad y están sujetos a acuerdos de procesamiento de datos que les prohíben usar los datos para fines distintos a la prestación del servicio.

La Empresa puede revelar datos cuando sea requerido por orden judicial o requerimiento legal válido de autoridad competente de la República del Paraguay, notificando al usuario afectado en la medida en que la ley lo permita.

6

Retención de datos

Los datos del usuario y los datos de auditoría procesados en la plataforma se retienen según el siguiente esquema:

  • Durante la suscripción activa: Todos los datos se conservan íntegramente y están disponibles para el usuario.
  • Tras la cancelación de la suscripción: Los datos permanecen disponibles durante 30 días adicionales para que el usuario pueda exportarlos. Transcurrido ese plazo, se eliminan de forma permanente e irrecuperable.
  • Datos de facturación: Se conservan por el período legalmente exigido según la normativa tributaria paraguaya (mínimo 5 años).
  • Logs de seguridad y audit trail: Se conservan por un mínimo de 12 meses por razones de seguridad e integridad del sistema.

El usuario puede solicitar la eliminación anticipada de sus datos enviando un email a hola@inteliaudit.com.py, lo cual implica la cancelación inmediata de la suscripción y la pérdida de acceso al servicio.

Derechos del usuario

El usuario tiene los siguientes derechos sobre sus datos personales, ejercibles en cualquier momento:

Acceso

Conocer qué datos personales tenemos almacenados sobre vos.

Rectificación

Corregir datos inexactos o incompletos desde la configuración de tu cuenta.

Eliminación

Solicitar la eliminación de tus datos y la cancelación de tu cuenta.

Portabilidad

Exportar tus datos y los de tus auditorías en formatos estándar (CSV, XLSX).

Oposición

Oponerte al tratamiento de tus datos para fines de comunicación comercial.

Limitación

Solicitar que limitemos el tratamiento de tus datos mientras se resuelve una reclamación.

Para ejercer cualquiera de estos derechos, enviá un email a hola@inteliaudit.com.py indicando el derecho que querés ejercer y tu identificación. Respondemos dentro de los 10 días hábiles.

8

Cookies y tecnologías de seguimiento

Inteliaudit utiliza un número mínimo de cookies, todas ellas estrictamente necesarias para el funcionamiento del servicio:

  • Cookie de sesión: Mantiene al usuario autenticado durante su navegación en la aplicación. Se elimina al cerrar el navegador o al cerrar sesión. No contiene datos personales, solo un token de sesión encriptado.
  • Cookie de preferencias: Almacena configuraciones básicas del usuario (idioma, zona horaria) para mejorar la experiencia. No se transmite a terceros.

Inteliaudit no utiliza cookies de tracking, cookies de publicidad, píxeles de seguimiento ni ninguna tecnología de análisis conductual de terceros (como Google Analytics u otras plataformas similares).

Sin tracking: No rastreamos tu comportamiento dentro de la aplicación para fines publicitarios ni compartimos datos de uso con plataformas de terceros. Los únicos datos de uso que recopilamos son logs técnicos para seguridad y diagnóstico de errores.

9

Seguridad de la información

Inteliaudit implementa un modelo de seguridad en capas:

  • Encriptación AES-256: Para todos los datos sensibles almacenados, incluyendo credenciales Marangatú, tokens de sesión y datos de auditoría marcados como confidenciales.
  • HTTPS obligatorio: Toda comunicación entre el cliente y el servidor usa TLS 1.3. No se sirve contenido sensible por HTTP sin cifrar.
  • Autenticación multifactor (MFA): Disponible y recomendada para todos los usuarios; obligatoria para administradores de cuenta Enterprise.
  • Audit trail inmutable: Registro completo de accesos y operaciones sobre datos de auditoría. Cada acción queda registrada con usuario, timestamp e IP, y no puede ser modificada ni por administradores.
  • Aislamiento de datos: Los datos de cada firma auditora están aislados mediante segmentación lógica de base de datos. Un usuario de una firma no puede acceder a los datos de otra firma bajo ninguna circunstancia.
  • Gestión de vulnerabilidades: Las dependencias de software se actualizan regularmente. Se realizan análisis de seguridad periódicos.

En caso de detectarse una brecha de seguridad que afecte datos personales, la Empresa notificará a los usuarios afectados dentro de las 72 horas de confirmada la brecha, detallando la naturaleza del incidente y las medidas adoptadas.
